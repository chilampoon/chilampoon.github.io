Units of RNA expression values

Chi Lam Poon
2019-03-25
RNA-seq

Keep in mind that three main steps in transciptomic anlysis are usually just:

  1. Alignments/mapping: assign reads to transcriptomics
  2. Read counting: estimate abundances
  3. Differential expression: compare abundances

So the units need to be noticed in abundance estimation and comparison. Note that expression feature is used to represent a genomic region, ie. gene, isoform, exon.

The expression units can be classified as raw counts and those after within sample normalization.

Counts

Without length adjustment, counts are not comparable within a sample, which means you can’t sum isoforms counts to get gene counts.

  • Effective counts

    Those adjusted for the amount of bias which is due to lower coverage in the ends of fragments.

  • Counts per million (CPM)

    Had been scaled by the number of fragments you sequenced times one million.

Within sample normalization: correct for feature length

Since the longer the length of a feature, the more the amount of counts, so after normalization of feature length, comparison within samples is allowed.

  • Transcripts per million (TPM)

    The proportion of transcripts in your pool of RNA, TPM is the number of transcripts you would have seen of type i, given the abundances of the other transcripts in your sample.

  • RPKM/FPKM

    Reads/Fragments per kilobase of exon per million reads mapped. FPKM == RPKM if you have single-end reads, pair-end uses FPKM. If you were to sequence this pool of RNA again, you expect to see FPKMi fragments for each thousand bases in the feature for every N/10^6 fragments you’ve sequenced.

    TPM can be calculated using FPKM.

Between sample normalization: correct for sequencing depth

BSN is often called library size normalization (LSN), while the library is not the cDNA in your sample after prep. After BSN, features are allowed to be compared across experiments.

Most BSN methods address two issues:

  1. Variable sequencing depth (the total number of reads sequenced) between experiments

  2. Finding a “control set” of expression features which should have relatively similar expression patterns across experiments (e.g. genes that are not differentially expressed) to serve as a baseline

The basic mathematical framework for BSN:

Observed counts for feature i from sample j (a sample from some distribution) follows the property: The expective value = mean expression rate for feature i × sample specific scaling factor for condition j

Some popular BSN methods:

  • TMM normalization
  • DESeq normalization
  • Upper quantile normaliztion
